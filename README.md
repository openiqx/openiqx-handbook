# OpenIQX Handbook

Welcome to the **OpenIQX Handbook**, the central guide for our community.

This repository contains all the knowledge and best practices for contributing to large open-source projects, mastering collaboration, and building a portfolio of real contributions.

---

## Purpose

- Document OpenIQX community philosophy and practices
- Provide step-by-step guides for beginners
- Share best practices for collaboration, Git, PRs, and codebase reading

---

## Structure

- `git-tips/` - Useful git how-tos for open source contributors (syncing forks, backporting, rebasing, and more)
- `handbooks/` - Core guides on collaboration, reading codebases, PR best practices, testing, and more
- `CONTRIBUTING.md` - Guidelines for contributing to all OpenIQX repos
- `CODE_OF_CONDUCT.md` - Community behavior and values

---

## Getting Started

1. Read the [CONTRIBUTING.md](CONTRIBUTING.md) for instructions on how to contribute.
2. Explore the `handbooks/` folder for guidance on working with large codebases.
3. Follow the step-by-step practices for claiming issues, writing commits, and submitting PRs.

---

## Notes

This is a **living handbook**. Updates are made as we learn new ways to collaborate and contribute faster to open-source projects.
